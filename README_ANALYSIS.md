## ⚠️ 데이터 품질 & 주의사항

### 결측치('-') 처리 규칙

#### 규칙 3-1
'광고세트' col이 '-'인 데이터는 '캠페인' col이 '-'인 데이터와 별도 데이터로 인식

#### 규칙 3-2
각 row를 봤을 때 결측치인 데이터는 다른 분류라고 판단해야함

#### 규칙 3-3
기기유형, 플랫폼, 연령, 성별이 '-' 값인 데이터이면, '기기플랫폼' 단일 데이터로 다뤄야함

#### 규칙 3-4
기기유형, 플랫폼, 기기플랫폼이 '-'값인 데이터고, '성별','연령' 데이터가 '-'가 아니라면,
'성별', '연령' combine 데이터이며 이때는 성별/연령 PIVOT 해서 판단 가능

### 구현 시 고려사항
- 각 데이터 타입별로 별도 집계 로직 필요
- PIVOT 분석 시 결측치 패턴 고려 필수
- 크로스 탭 분석 시 유효 차원만 사용
