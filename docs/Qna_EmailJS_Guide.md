# Qna.html - EmailJS 이메일 전송 기능 가이드

## 변경 개요

기존 `mailto:` 링크 방식에서 **EmailJS API**를 사용한 직접 이메일 전송 방식으로 변경

| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| 전송 방식 | `mailto:` 링크 (메일 앱 실행) | EmailJS API (직접 전송) |
| 사용자 경험 | 메일 앱에서 수동 전송 필요 | 버튼 클릭만으로 전송 완료 |
| 수신 이메일 | support@growthmaker.kr | tee1228@naver.com |

---

## 기능 설명

### 전송 흐름
1. 사용자가 문의 폼 작성
2. '문의 저장하기' 버튼 클릭
3. 버튼 비활성화 + "전송 중..." 표시
4. EmailJS API로 이메일 전송
5. 성공 시: 성공 메시지 표시 + 폼 초기화
6. 실패 시: 에러 알림 표시
7. 버튼 복원

### 폼 필드
| 필드명 | ID | 필수 여부 |
|--------|-----|-----------|
| 회사명 | `companyName` | 필수 |
| 담당자명 | `contactName` | 선택 |
| 이메일 | `contactEmail` | 선택 |
| 문의 유형 | `inquiryType` | 필수 |
| 문의 내용 | `inquiryContent` | 필수 |

## 제약 사항

### 기술적 제약
1. **인터넷 연결 필수**: EmailJS API 호출을 위해 인터넷 연결 필요
2. **EmailJS 서비스 의존성**: EmailJS 서버 장애 시 전송 불가
3. **월간 전송 제한**: EmailJS 무료 플랜 기준 월 200건 제한

### 보안 고려사항
1. Public Key가 클라이언트에 노출됨 (EmailJS 특성상 정상)
2. 스팸 방지를 위해 EmailJS 대시보드에서 도메인 제한 설정 권장
3. reCAPTCHA 연동 고려 가능

### 브라우저 호환성
- 모던 브라우저 지원 (Chrome, Firefox, Safari, Edge)
- ES6 Promise 지원 필요
